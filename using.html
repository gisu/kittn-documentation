Setup

After the installation is complete, the project must be initialized.


gulp init

Init Process:

  • Build a individual Modernizr Build and Copy to dist
  • Copy System Files to dist
  • Copy JS Files to dist
  • Copy HTML Images from src/stash/images/html-images/html-assets/ to dist/images
  • Start the grunt Build Image Task
    • Copy to dist/assets/img/sprite/
    • Copy to dist/assets/img/sprite_2x/
    • Build Retina Sprite src/sass/maps/
    • Build NonRetina Sprite src/sass/maps/
    • Rebuild Retina Sprite Map dist/assets/img/
    • Rebuild NonRetina Sprite Map dist/assets/img/
    • SVG Minify and Copy to dist/assets/img/svg/
    • Build PNG Fallback Image dist/assets/img/svg
    • SVG Minify and Copy to dist/assets/img/svgonly/
    • Copy Texture Image to dist/assets/img/texture/
    • Copy Single Images to dist/assets/img/single/
    • Build Image Map src/sass/maps/
  • Compile Sass Files from src/sass/ into dist/assets/css/
  • Compile Jade Files from src/jade/ into dist/
  • Check JS File from src/js/ and copy it to dist/assets/js/
  • Combine some JS Files and copy it to dist/assets/js/scripts.js

When initializing, different files from scr directory will be copied to the distribution dist directory. In addition, js files are combined, html and css would be compiled.

Important! You will always work in the src directory. There, all source data are stored, which are then processed on Gulp task on.

In order now to begin the work activate the default task:


gulp

Gulp will activate the server that provides a preview in realtime on the browser - the browser is automatically refreshed after every change on the filesystem. Within the src directory js, Jade and Sass files are monitored for a change in the compiler is active - in the case of Javascript JSHint will check code quality.

I deliberately use no CoffeeScript because it has just performed in teamwork to confusion. If you still want to use CoffeeScript, only the corresponding plugin must install.

Those who want to extend the file monitoring to images, can use the extended default task:

bash gulp extended

A modification of files, as well as adding or deleting will activate the task, eg is from SVG files automatically generates a PNG fallback.

It may be that the Extend task is not responding at the beginning of changes in the image (src/stash/images/) directory. Since no images at the beginning exist that can Gulp watch. Preparing in this case the images on gulp grunt-rebuild-images and then activate the extended task.

  • On File Change

    • Clean Texture Directory dist/assets/img/sprite_2x/

Tasks

gulp init<br> For the first initialization of the project.

gulp<br> Default Task

gulp extended<br> Extended default task - monitors image data

gulp rebuild-js<br> All the Javascript files are rebuilded (combined, copied, checked)

gulp publish<br> The project data is prepared for production. CSS, JS and image data are compressed. The documenation with sassdoc is generated.

gulp jade-rebuild<br> Rebuild all Jade Files

gulp bump<br> Patch the Version Number from package.json and bower.json (0.0.1)

gulp bump-minor<br> Extend the Minor Version Number from package.json and bower.json (0.1.0)

gulp bump-major<br> Extend to the next Release Number from package.json and bower.json (1.0.0)

gulp grunt-rebuild-images<br> Rebuild all Image Files.

gulp grunt-responsive-images<br> Build different Sizes from HTML Images

Structure

As already explained, one will usually always work in src directory.

root<br> .. dist/ distribution<br> .. src/ source

The internal structure of the dist / directory looks like this:

dist/<br> .. assets/<br> .... css/ CSS files<br> .... img/ CSS included images<br> ...... system/ Systemimages like favicon<br> ...... single/ CSS single images<br> ...... textures/ CSS texture images<br> .... js/ JS files<br> .... fonts/ Webfonts & iconfonts<br> .. images/ HTML included images

If you therefore should not be satisfied, you can change the structure of the package.json:

If the paths are changed, the paths must be in the Sass setup file (src/sass/_setup.scss -> $kittn-directorys) also be changed accordingly

This should happen before the gulp init of course, otherwise you have to rename the directories manually, or you delete dist simple and then starts the gulp init again.

The internal structure of the src / directory:

src/ <br> .. jade/ Jade files<br> .. js/ JS files<br> .. markdown/ External texts in Markdown format<br> .. sass/ Sass framework<br> .. stash/ filing<br> .... .sytem/ Internal system files<br> .... bower/ Bower dependencies<br> .... fonts/ Font files<br> .... images/ Image sources<br> ...... css-images/ CSS included images<br> ........ single-assets/ Place for each single assets<br> ........ sprite-assets/ Place for each sprite assets<br> ........ svg-assets/ Place for each svg assets with a generated png fallback<br> ........ svgonly-assets/ Place for each svg assets without fallbacks<br> ........ texture-assets/ Place for each texture asset<br> ...... html-images/<br> ........ html-assets/ HTML included images<br> .... scripts/ Script files that are not installed on Bower<br>
